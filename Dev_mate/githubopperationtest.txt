================================================================================
 PREREQUISITES (Must Do Before Testing)
================================================================================

  Before testing GitHub operations, complete these setup steps:

  STEP 1: Verify Git is installed
  ────────────────────────────────
      git --version
      Expected: git version 2.x.x or higher

  STEP 2: Install GitHub CLI (gh)
  ────────────────────────────────
      Download from: https://cli.github.com/
      Verify:  gh --version
      Expected: gh version 2.x.x or higher

  STEP 3: Authenticate GitHub CLI
  ────────────────────────────────
      gh auth login
      → Select "GitHub.com"
      → Select "HTTPS"
      → Authenticate with your browser or token
      → Verify:  gh auth status

  STEP 4: Configure Git identity
  ────────────────────────────────
      git config --global user.name "Your Name"
      git config --global user.email "your.email@example.com"

  STEP 5: Launch DevMate
  ───────────────────────
      cd F:\project_folders\DevMate_backup
      python devmate.py

================================================================================
 TEST 3: PUSH TO GITHUB (Auto-Flow)
================================================================================

  This test checks the full automatic flow:
  DevMate will auto-detect your last project, init git if needed,
  commit files, and create a GitHub repo.

  TYPE IN DEVMATE (any of these):
  ─────────────────────────────────
      Push my project to GitHub
      Push to GitHub
      Push my code to GitHub
      Git push
      Push code to remote

  EXPECTED FLOW:
      1. DevMate finds your last created project from memory
      2. If no .git folder exists → runs "git init" automatically
      3. Stages all files → runs "git add ."
      4. Commits → runs "git commit -m 'Update via DEVMATE'"
      5. Tries "git push origin main"
      6. If no remote → creates a GitHub repo via "gh repo create"
      7. Final message: ✅ GitHub repo '<name>' created and pushed.

  VERIFY ON GITHUB:
      - Go to https://github.com/YOUR_USERNAME
      - You should see a new repo called "GitHubTestProject"
      - Click on it → files should match your local project

================================================================================
 ERROR SCENARIOS & TROUBLESHOOTING
================================================================================

  ERROR: "Git commit failed"
  ──────────────────────────
  CAUSE: No files to commit (nothing changed since last commit)
  FIX:   Make a change to any file, then try again

  ERROR: "git push failed"
  ────────────────────────
  CAUSE: No remote configured, or wrong branch name
  FIX:   DevMate auto-creates a GitHub repo as fallback.
         If that also fails, run manually:
           gh auth login
           cd <project-path>
           gh repo create <name> --public --source . --push

  ERROR: "gh repo create failed"
  ──────────────────────────────
  CAUSE: GitHub CLI not installed or not authenticated
  FIX:
    1. Install GitHub CLI:     https://cli.github.com/
    2. Authenticate:           gh auth login
    3. Verify:                 gh auth status

  ERROR: "Command not found: git"
  ────────────────────────────────
  CAUSE: Git is not installed or not in PATH
  FIX:   Install Git from https://git-scm.com/
         Restart your terminal after installation

  ERROR: "Command not found: gh"
  ───────────────────────────────
  CAUSE: GitHub CLI is not installed or not in PATH
  FIX:   Install from https://cli.github.com/
         Restart your terminal after installation

  ERROR: "Project path not found"
  ────────────────────────────────
  CAUSE: No project was created yet, or the project was deleted
  FIX:   Create a project first:
         "Create a Python project called MyProject"
